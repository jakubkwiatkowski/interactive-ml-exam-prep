Essential definitions

Effect size

The magnitude of a difference or relationship, separate from statistical significance. Examples: an accuracy difference of 0.03, Cohen’s \(d\), an odds ratio, or rank-biserial correlation.

Statistical power

The probability of rejecting \(H_0\) when a specified real effect exists: \(Power=1-\beta\). It increases with larger effects, more independent observations, lower noise, and valid informative assumptions.

Parametric method

Uses a model with parameters and stronger assumptions, often about means, variances, and distributional form.

Non-parametric method

Avoids a specific parametric distributional model, often using signs, counts, or ranks. It is not assumption-free.

Commensurability

Whether numerical differences have comparable meaning across observations or domains.

Paired design

Both methods are evaluated under the same condition: same cases, split, or dataset.

Omnibus test

A global test asking whether any difference exists among several methods.

Post-hoc test

A follow-up after an omnibus rejection, identifying which comparisons differ while controlling multiplicity.

i.i.d.

Independent and identically distributed.

Effort / review capacity

The number or fraction of predictions that can be inspected or acted upon.

Evaluation questions are different

Performance estimation

How well does the model or learning procedure generalize?

Significance testing

Is an observed difference larger than expected under a null model?

Practical importance

Is the effect large enough to matter operationally?

Do not merge them: a p-value is not an effect size, and statistical significance is not practical importance.

Why parametric tests can be more powerful: they use how large differences are, not just their direction or ordering.
Wilcoxon and commensurability: Wilcoxon still assumes absolute differences can be ordered meaningfully, but it does not require exact ratios such as 0.04 being exactly twice 0.02.

Core tests — compact notes

\[t=\frac{\bar d}{s_d/\sqrt n}.\]

Use when pairs are independent across observations, differences are approximately normal for small \(n\), and the numerical scale is meaningful. Normality concerns \(d_i\), not the two score columns separately.

Ordinary CV fold scores are dependent because training sets overlap. A naive paired t-test over folds can underestimate uncertainty.
Almost no assumptions about numerical scale; only direction matters.
A tiny win and a huge win count equally. For the usual multi-dataset sign test, one trial is one dataset, not one fold.

Uses signed ranks of \(|d_i|\). It is less sensitive to outliers and heterogeneous scales than paired t, while using more information than the sign test.

It is not literally the same hypothesis as “mean difference equals zero.”
Repeated-measures ANOVA

Raw scores, normal residual structure, and sphericity: equal variances of pairwise classifier differences.

Friedman

Ranks classifiers within each dataset. Datasets are independent blocks; folds remain inside each dataset-level score.

Friedman is safer across heterogeneous domains, but it discards magnitude.

Positive correlation means repeated splits contain less independent information than their nominal number suggests. The naive standard error omits this dependence and is too small.

NHST exam insights

Support, not proof

Rejecting at \(\alpha=.05\) does not mean a 95% probability that the alternative is true.

Significance ≠ importance

Report effect size and actual score differences.

Test choice defines the null

Mean difference, win probability, disagreement symmetry, and equal ranks are different questions.

Use the same splits

Pairing removes split-to-split noise.

Do not count significant wins only

Thresholding each domain at \(p<.05\) discards consistent weak evidence.

Dataset selection matters

Inference applies to the population represented by the benchmark suite.

More datasets improve power

For Wilcoxon/Friedman, \(N\) is the number of independent datasets.

Omnibus then post-hoc

Use all-vs-all or control-vs-rest according to the question.

Non-rejection is not equality

It may mean low power or noisy estimates.

Ranks trade magnitude for robustness

They reduce outlier/scale sensitivity but lose numerical detail.

ROC, PR, AUC and balanced metrics

ROC

Plots \(TPR\) versus \(FPR\) as threshold changes.

PR

Plots precision versus recall and exposes positive-alert trustworthiness.

AUC

Probability that a random positive is ranked above a random negative.

\[AUC=P(s(X^+)>s(X^-)).\]
AUC is not simply an average over uniformly spaced thresholds; ROC integration weights intervals in \(FPR\).
ROC advantages

Prevalence-insensitive class-conditional rates and symmetric class treatment.

ROC weaknesses

A small FPR can still mean many false alerts, and alert trustworthiness is hidden.

PR advantages

Useful for retrieval, fraud, rare disease, and object detection.

PR weaknesses

Depends on prevalence, ignores TN, and is not directly comparable across different prevalences.

\[Precision=\frac{\pi\,TPR}{\pi\,TPR+(1-\pi)FPR}.\]
Same ROC point, different precision: with \(TPR=.8,FPR=.01\), precision is about .988 at \(\pi=.5\), but about .074 at \(\pi=.001\).
Youden’s \(J\) and balanced accuracy
\[J=TPR-FPR=TPR+TNR-1,\qquad Balanced\ Accuracy=\frac{J+1}{2}.\]

Maximizing \(J\) gives equal importance to positive and negative classwise performance. This need not match unequal deployment costs.

Cost and threshold — intuitive guide

ROC asks

What \(TPR/FPR\) trade-offs are available?

PR asks

How trustworthy are positive alerts as recall changes?

Cost curves ask

For which prevalence-and-cost conditions is each threshold cheapest?

\[EC(t)=P(+)C_{FN}FNR(t)+P(-)C_{FP}FPR(t).\]
\[PCF=\frac{P(+)C_{FN}}{P(+)C_{FN}+P(-)C_{FP}}.\]
Near \(PCF=0\), false-positive burden dominates. Near \(PCF=1\), false-negative burden dominates.
\[NEC(t)=FNR(t)PCF+FPR(t)[1-PCF].\]

For one fixed threshold, \(FNR\) and \(FPR\) are constants, so \(NEC\) is a straight line. The lower envelope of all threshold lines is the cheapest available decision for each operating condition.

Sampling: model versus estimator bias and variance

ObjectBiasVariance
Model / learning algorithmAverage prediction systematically differs from optimal prediction.Fitted function changes with the training sample.
Error estimatorExpected reported error differs from its target.Reported error changes across samples/splits.
Small train, large test

Potentially worse/more variable model; lower test-sampling variance; possible pessimism relative to a full-data final model.

Large train, small test

Model closer to full-data fit; smaller training-size mismatch; high variance of the error estimate.

A biased or poor model can still have its own risk estimated accurately. Model bias does not automatically imply estimator bias.
\[Var(\widehat R)=E_S[Var(\widehat R\mid S)]+Var_S(E[\widehat R\mid S]).\]

The first term is test-sample randomness; the second comes from training-sample/model variation.

Why the tutorial statements are not contradictory

Bias is defined after averaging over possible training samples of a specified size. Variance measures variation across those samples. Changing sample size changes the distribution of training samples and can change both bias and variance.

Leave-one-out: the subtle case

Training-size bias

Low, because every model trains on \(n-1\approx n\).

Dispersion among LOO models

Often low because training sets almost coincide.

Variance of the final estimate

Can still be high because fold losses are dependent.

\[\widehat R_{LOO}=\frac1n\sum_{i=1}^n L_i.\]

The \(L_i\)'s are not independent, so averaging them reduces uncertainty less than averaging \(n\) independent losses.

“Practically unbiased classifier” means little training-size mismatch, not zero model bias.
LOO is mainly for small-to-moderate \(n\) with cheap training. Tiny samples can still be unstable; large samples make LOO too expensive.

Choosing an error-estimation method

Domain / dependence

Preserve patients, times, devices, groups, and locations.

Dataset size

Large data can afford untouched hold-out; small data often needs CV/bootstrap.

Bias–variance

Small training folds can be pessimistic; small test folds increase estimator variance.

Expected error

Very low error requires enough test cases to observe failures.

Algorithm type

Bootstrap duplicates act like weights; unstable/order-sensitive algorithms react to resampling.

Computational cost

Hold-out fits once; \(k\)-fold fits \(k\); 10×10 fits 100; LOO fits \(n\); bootstrap fits \(B\).

Repeated CV: averages arbitrary fold construction and shows partition sensitivity, but does not create new population information.

Final compact summary

Significance

Choose the test from the statistical unit and null hypothesis; report effect size.

Metrics

ROC describes class-conditional ranking; PR describes positive-alert quality; cost curves add prevalence and consequences.

Sampling

Always state whether bias/variance belongs to the model, point estimator, or uncertainty estimator.

Deployment

The split, metric, threshold, and cost model must match the operating condition.
