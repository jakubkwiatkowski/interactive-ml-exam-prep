1. The main idea: sampling and significance testing are linked

The strongest idea from the tutorial is that the statistical test cannot be chosen independently of the evaluation design. First you decide how to estimate performance (hold-out, cross-validation, repeated CV, fixed test set, many datasets), and only then can you decide what kind of statistical evidence you actually have.

Step 1

Scientific question

"Is my new classifier better?" or "Do these algorithms differ at all?"

Step 2

Evaluation design

One dataset or many? Fixed test set or repeated resampling? Two classifiers or many?

Step 3

Error estimation

How did each score arise? Hold-out, k-fold CV, repeated CV, 5x2cv, bootstrap, etc.

Step 4

Statistical unit

What counts as one observation: one example, one split/run, or one dataset?

Step 5

Significance test

Choose a test whose target and assumptions match the design and statistical unit.

Inside one dataset

Cross-validation mainly helps you estimate a reliable score on that dataset. But fold results are usually dependent, because training sets overlap.

Across many datasets

In Demšar-style comparisons, the final statistical sample is the set of datasets. Cross-validation is used inside each dataset only to estimate a score.

Exam trap

"I used 10 datasets and 10-fold CV, so I have 100 independent observations." Usually false. For Wilcoxon/Friedman, the sample size is often the number of datasets.

Guiding question

What is one observation for the test? Until you can answer that, do not choose a significance test.

4. One observation? This is often the whole exam.

Many errors in ML evaluation come from mixing levels. The table below is meant to make that distinction explicit.

Scenario What is one observation for the statistical test? What produces each number? Typical tests Main trap
Two classifiers on one fixed test set One test example / one instance / one case Paired correctness of both classifiers on the same example McNemar Thinking a “case” means a whole dataset. Here it means one test instance.
Two classifiers on one dataset with repeated splits One repeated split / run (but note dependence issues) Repeated train/test performance estimates Corrected resampled t-test, 5x2cv t or F Treating repeated splits as fully independent without correction.
Two classifiers on one dataset with k-fold CV Fold-level scores are not safely independent CV fold performance values Be careful; avoid naive paired t-test over folds Using a plain t-test over the k folds as if the folds were i.i.d.
Two classifiers over multiple datasets One dataset-level difference One reliable performance score per dataset per classifier (often from CV) Sign test, Wilcoxon signed-rank Multiplying by the number of folds and pretending you have more datasets.
Many classifiers over multiple datasets One dataset with ranks of all classifiers Performance table → rank each row Friedman, then Nemenyi / Bonferroni-Dunn / Holm Skipping the omnibus test or doing many uncorrected pairwise tests.

5. Parametric vs non-parametric: intuitive comparison

Parametric tests make stronger assumptions about the distributional form of the data or statistic. Non-parametric tests usually rely on fewer assumptions and often work with counts, signs, or ranks.

Parametric

  • Examples: paired t-test, repeated-measures ANOVA, corrected resampled t-test, 5x2cv t/F.
  • Typical target: mean difference or variance decomposition.
  • Typical assumptions: normality (often of differences), meaningful numerical scale, sometimes homogeneity/sphericity.
  • Strength: can be more powerful when assumptions are truly reasonable.
  • Risk: if assumptions are wrong, p-values can mislead you.
Intuition: you trust the actual numeric values enough to model their behavior more strongly.

Non-parametric

  • Examples: McNemar, sign test, Wilcoxon signed-rank, Friedman, Nemenyi.
  • Typical target: wins/losses, signs, ranks, rank differences.
  • Typical assumptions: usually weaker; often only independence of the chosen observations and sensible ordering/ranking.
  • Strength: more robust when numerical differences are not safely commensurable or normality is doubtful.
  • Risk: sometimes less powerful, and sometimes it answers a slightly different question than the mean-difference question.
Intuition: you give up some numerical detail to gain robustness and safety.

6. Exam-style self-check

Use these as quick oral-exam drills. Try answering before revealing the solution.

Q1. I have 15 datasets and on each I used 10-fold CV. What is the sample size for Wilcoxon?

Usually 15, not 150. The 10 folds help estimate one performance value per dataset. Wilcoxon compares the 15 paired dataset-level differences.

Q2. In McNemar’s test, what is one case?

One test example / instance from the same practical dataset. Example: one patient, one email, one image. The test compares how the two classifiers behave on that example.

Q3. Why is naive paired t-test over CV folds dangerous?

Because folds are not safely independent as observations for the test. The training sets overlap, so the variance may be underestimated and Type I error inflated.

Q4. Friedman is significant, but Nemenyi finds no pairwise differences. Contradiction?

No contradiction. Friedman is an omnibus test: it says some difference exists. A post-hoc test may still lack enough power to identify which exact pairs differ.

7. Final exam mental checklist

Ask 1: What is one observation?

One test example? One repeated split? One dataset? This decides the whole test family.

Ask 2: What does “no difference” mean here?

Equal disagreement counts? Zero mean difference? Equal win probability? Equal ranks?

Ask 3: Are the observations independent enough?

Especially important for repeated splits and CV folds.

Ask 4: One dataset or many?

Single-domain tests and multiple-dataset tests answer different scientific questions.

Ask 5: Two classifiers or many?

Pairwise tests are not a substitute for an omnibus test when many classifiers are compared.

Ask 6: Is this all-vs-all or vs a control?

That decides whether Nemenyi or control-focused procedures like Bonferroni-Dunn/Holm are more appropriate.
