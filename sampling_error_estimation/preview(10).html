How to choose an error-estimation method

These are the practical criteria from the tutorial, expanded with precise meanings.

Domain / dependence structure

Use random IID splits only if examples are approximately independent. For time series, split chronologically. For patient/user/group data, split by group. For spatial data, consider location-based splits.

Dataset size

Large datasets can often afford a clean hold-out set. Small datasets often need CV or bootstrap because throwing away a large test set leaves too little training data.

Bias–variance dependence

Small training sets can make the learned models worse: this is training-size bias relative to a full-data model. Small test sets make the error estimate noisy: this is variance of the evaluation estimate, not necessarily model variance.

More criteria

Expected true error rate: if error is very low, small test sets may observe zero errors and look overconfident. If error is high, optimistic methods like resubstitution are especially misleading.

Algorithm type: bootstrap duplicates act like sample weights; some algorithms are sensitive to duplicates/order/permutation.

Computational complexity: LOO trains \(n\) models, 10×10CV trains 100 models, bootstrap can train hundreds or thousands.
