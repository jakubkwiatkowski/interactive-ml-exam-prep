The central relationship: the split defines the evidence

A performance number is not produced only by a metric. It is produced by a full experimental design. The same classifier and the same metric can give different meanings under hold-out, cross-validation, bootstrap, temporal validation, or group validation.

1Target population

What future examples, patients, times, locations, or domains do we care about?

2Available sample

One finite dataset drawn imperfectly from that target population.

3Split / resampling

Decide which observations train models and which imitate future unseen data.

4Train model(s)

One model for hold-out; \(k\) models for \(k\)-fold CV; many models for repeated CV/bootstrap.

5Error estimator

Average loss on test-like observations or average across folds/repetitions.

6Interpretation

Specify what was estimated, uncertainty, dependencies, and deployment scenario.

Golden exam question: What exactly is the target of this estimate: one fixed trained classifier, a learning algorithm trained on a particular sample size, or a future final model trained on all available data?
One fixed hold-out model

Estimate error of the specific model trained on \(S\), using untouched test sample \(T\).

Cross-validation

Estimate average behavior of a learning procedure trained repeatedly on \((k-1)/k\) of the data.

Structured validation

Estimate future-time, unseen-person, unseen-group, or unseen-location generalization.

Terminology map: sample, estimator, estimate, error and risk

Many confusions disappear once every noun is attached to the correct object.

Population / target distribution \(D\)

The conceptual source of future examples. It may mean future IID cases, future patients, future times, or new locations—depending on deployment.

Available dataset \(D_n\)

The finite observed sample. It is not the population itself and does not contain all possible future variation.

Training sample \(S\)

The observations used to fit parameters, choose rules, or learn representations.

Test sample \(T\)

Observations kept outside fitting and used to imitate unseen cases.

Sampling / splitting method

The rule that assigns observations to train/test roles: hold-out, folds, bootstrap draws, time blocks, groups.

Resampling

Reuse the finite dataset in multiple train/test arrangements. It does not create new real information.

True risk \(R(f)\)

The unknown expected loss on the target population.

Estimator

A random rule/formula computed from a sample, e.g. test error \(R_T(f)\).

Estimate

The concrete numerical result produced by an estimator, e.g. \(15/100=0.15\).

Bias of an estimator

Systematic difference between its expected value and its target quantity.

Variance of an estimator

How much the reported estimate changes across possible samples/splits.

Model variance

How much the learned prediction function changes when the training sample changes. Do not confuse this with variance of the error estimate.

Vocabulary trap: “sample” can mean the whole observed dataset, one train/test subset, one bootstrap dataset, or—as a verb—one draw. Always name the level explicitly.

Taxonomy clarification: ordinary \(k\)-fold CV already contains multiple train/test runs. In the tutorial’s wording, “simple resampling” means one complete resampling design performed once, while “multiple resampling” repeats an entire design or generates many bootstrap/randomization samples.

Why is resampling needed?

With one finite dataset, training quality and evaluation precision compete for the same observations.

Too little training data

The trained model may be systematically worse than a model trained on more data. This is a model/training-size issue.

Too little test data

The measured error fluctuates strongly from one possible test sample to another. This is variance of the error estimator.

Test observations

Training-data pressure
Resampling does not remove the finite-data limitation. It allows each observation to serve as test evidence in some runs and training information in other runs, making more efficient use of the available sample.
What did the tutorial mean by “allows for closer duplication conditions”?

It is best read as: resampling more closely imitates repeated experiments. Ideally, we would collect many fresh datasets from the population, retrain, and retest. Since we usually have only one dataset, repeated splits/bootstrap samples approximate those repeated conditions.

TRAIN = TESTOPTIMISTICONE MODEL
ProcedureTrain on all available observations and evaluate on those same observations.
What is estimated?Training error—not clean unseen-data/generalization error.
Why optimistic?The fitted model was selected specifically to perform well on those observations; flexible models can memorize them.
Useful forDiagnostics, optimization monitoring, and as the optimistic component of the .632 bootstrap.

Complete overfit: a sufficiently flexible classifier may achieve almost zero re-substitution error while failing badly on new examples.

Hold-out evaluation

INDEPENDENT TEST SETONE MODELCLEAN CONFIDENCE BOUNDS
ProcedureSet aside test set \(T\); train \(f_S\) on \(S\); evaluate only once on untouched \(T\).
What is estimated?Risk of the specific classifier trained on \(S\). Relative to a future model trained on all data, the estimate may be pessimistic because \(S\) is smaller.
AdvantagesIndependence from training, simple interpretation, model-agnostic, clean binomial/Gaussian/concentration uncertainty analysis.
Main weaknessOne split can be unrepresentative; small \(T\) gives high estimator variance; small \(S\) can produce a weaker model.
Best whenDataset is large enough to afford a substantial untouched test set.
True risk
\[ R(f)=\mathbb{E}_{(x,y)\sim D}\left[L(y,f_S(x))\right]. \]
Empirical hold-out risk
\[ R_T(f)=\frac{1}{m'}\sum_{i=1}^{m'}L(y_i,f_S(x_i)). \]
“Generalization behavior can be characterized”: because \(T\) is outside fitting, its losses provide evidence about unseen cases. This does not mean zero uncertainty; it means the evidence is not training-contaminated.
“Estimates for any classifier”: hold-out is model-agnostic. It needs only predictions/losses, not differentiability, a probabilistic model, or access to internal parameters.

One example: Bernoulli

\(Z_i=1\) for an error and \(0\) for correct. \(Z_i\sim Bernoulli(p)\), where \(p=R(f)\).

Error count: Binomial

\(K=\sum_i Z_i\sim Binomial(m',p)\). Its mean is \(m'p\), not \(p\).

Error rate: sample mean

\(\hat p=K/m'\). Its expectation is \(p\), and CLT gives a Gaussian approximation for large \(m'\).

\[ \mathbb{E}[K]=m'p,\qquad \mathbb{E}\!\left[\frac{K}{m'}\right]=p,\qquad R_T(f)=\hat p=\frac{K}{m'}. \]
Exact binomial inversion

Observe \(K=k\), then find which values of \(p\) would make this count plausible. Produces lower/upper binomial confidence bounds.

Gaussian / CLT

Approximate the sampling distribution of \(\hat p\) as normal and use \(\hat p\pm zSE\).

Hoeffding

Use only independence and bounded loss \(Z_i\in[0,1]\). Safe and distribution-free, but usually conservative.

\[ CI_{\mathrm{Gauss}} =\hat p\pm z_{1-\delta/2}\sqrt{\frac{\hat p(1-\hat p)}{m'}}. \]
\[ P\!\left(|R_T(f)-R(f)|>\epsilon\right)\le 2e^{-2m'\epsilon^2}. \]
The displayed tutorial formula has Hoeffding form. It is not the ordinary Gaussian interval, even though the surrounding slides discuss Bernoulli/binomial errors.
\[ P(|R_T-R|>\epsilon)\le 2e^{-2m'\epsilon^2}. \]

Demand failure probability at most \(\delta\):

\[ 2e^{-2m'\epsilon^2}\le\delta \Longrightarrow e^{-2m'\epsilon^2}\le\frac{\delta}{2} \Longrightarrow -2m'\epsilon^2\le\ln\!\left(\frac{\delta}{2}\right). \]
\[ 2m'\epsilon^2\ge\ln\!\left(\frac{2}{\delta}\right) \Longrightarrow \epsilon=\sqrt{\frac{1}{2m'}\ln\!\left(\frac{2}{\delta}\right)}. \]
\[ \epsilon^2=\frac{1}{2m'}\ln\!\left(\frac{2}{\delta}\right) \Longrightarrow 2m'\epsilon^2=\ln\!\left(\frac{2}{\delta}\right) \Longrightarrow m'\ge\frac{1}{2\epsilon^2}\ln\!\left(\frac{2}{\delta}\right). \]
Final meaning: \(m'\) is the required independent test-set size—not training size and not total dataset size.
\(\epsilon\)Allowed absolute difference between observed test error and true risk. \(\epsilon=.05\) means ±5 percentage points.
\(\delta\)Failure probability. \(\delta=.05\) means the bound is intended to succeed with probability at least \(1-\delta=.95\).
\(m'\)Number of independent hold-out test observations contributing to the empirical average.
GrowthRequired \(m'\) grows as \(1/\epsilon^2\). Halving the desired margin requires roughly four times as many test examples.

k-fold cross-validation

ONE COMPLETE RESAMPLING DESIGNUSES ALL CASES FOR TESTINGOVERLAPPING TRAINING SETS
ProcedurePartition into \(k\) folds. For each fold, train on the other \(k-1\) folds and test on the held-out fold.
Models trained\(k\).
Training size per modelApproximately \(n(k-1)/k\).
What is estimated?Average behavior of the learning algorithm trained on \((k-1)/k\) of the data—not risk of one fixed classifier.
Main advantageEvery case receives an out-of-fold prediction while each model trains on most data.
Main complicationTraining sets overlap heavily, so fold estimates are dependent.
The fold test sets are disjoint within one CV run, but the fitted models are not independent because their training sets share most observations. This is why a naive ordinary t-test over fold scores can underestimate uncertainty.
Why can CV be biased?

The CV models train on fewer observations than a final model trained on all \(n\) cases. If learning improves with training size, CV can be pessimistic relative to the final full-data model. Small \(k\) creates larger training-size mismatch. Large \(k\) reduces that mismatch but increases overlap and computational cost.

Why “averaged estimate over \(k\) classifiers” matters

Each fold has its own fitted classifier \(f_1,\ldots,f_k\). The reported CV score averages their test losses. After CV, you usually fit a new final classifier on all data; that final classifier was not directly tested by the CV folds.

Your earlier intuition was correct: slight imbalance in a large originally balanced dataset is often less damaging. In an imbalanced dataset, a small change can remove most or all minority examples from a fold, making evaluation unstable or undefined.

Leave-one-out cross-validation (LOO)

\(k=n\)TRAIN ON \(n-1\)TRAIN \(n\) MODELS
ProcedureFor each observation, train on all other \(n-1\) observations and test on that single case.
Training-size biasSmall relative to a model trained on all \(n\), because \(n-1\approx n\).
Estimator varianceCan be high: each individual test fold contains one observation, and fitted models are extremely correlated.
Computation\(n\) model fits; too expensive for large \(n\) unless the algorithm has an efficient shortcut.
Possible useModerate dataset sizes, cheap learning algorithms, situations where losing training observations is costly.
“Practically unbiased classifier” in the tutorial: interpret this narrowly as little training-size mismatch. It does not mean the model has zero modeling bias or that the finite sample suddenly reveals the true data-generating function.
Why can LOO still have high variance?

LOO models differ by only one observation, so their errors can be highly correlated. A single influential/extreme observation can strongly affect the aggregate. “Large training sets per fold” and “low variance of the final estimator” are not the same statement.

Wide dispersion and extreme values

The tutorial notes potential usefulness under wide dispersion/extreme values because every case gets tested individually and almost all other observations train the model. However, influential outliers can also make LOO unstable, so this is not a universal guarantee.

Bootstrap, \(E_0\), and the .632 estimate

SAMPLING WITH REPLACEMENTOUT-OF-BAG TESTINGMANY MODELS
Core ideaTreat the empirical dataset as an approximation to the population and simulate repeated samples by drawing with replacement.
One bootstrap sampleDraw \(n\) positions from \(n\) observed cases with replacement. Size is \(n\), but unique cases are about \(0.632n\).
Out-of-bag (OOB)Original cases not selected in that bootstrap sample—about \(0.368n\)—act as test cases.
Does it create information?No. It simulates sampling variability using the empirical distribution.
Main assumptionThe observed sample is representative enough that its empirical distribution approximates the target population.
Bootstrap positions

.632 error

For one original observation, probability of not being selected on one draw is \(1-1/n\). After \(n\) draws:

\[ P(\text{not selected})=\left(1-\frac{1}{n}\right)^n\longrightarrow e^{-1}\approx0.368. \]
\[ P(\text{selected at least once})\approx1-0.368=0.632. \]
“After \(n\) samples” in the slide means after \(n\) draws within one bootstrap dataset—not after \(n\) bootstrap repetitions.
  1. For \(b=1,\ldots,B\), draw bootstrap training sample \(D_b^*\) of \(n\) positions with replacement.
  2. Train model \(f_b\) on \(D_b^*\).
  3. Test on OOB set \(O_b\).
  4. Compute \(e_{0b}\), then average.
\[ E_0=\frac{1}{B}\sum_{b=1}^{B}e_{0b}. \]
\(E_0\) can be pessimistic relative to a final model trained on all unique data because each bootstrap model sees only about 63.2% unique cases.

Compute resubstitution error—usually once—then combine it with the repeated OOB estimate:

\[ E_{.632}=0.368E_{\mathrm{resub}}+0.632E_0. \]
Repeated many times: bootstrap/OOB component. Usually computed once: full-data resubstitution error. The method combines optimistic training error with often pessimistic OOB error.

Bootstrap is attractive when a permanent hold-out would leave too few observations for both training and testing. It generates many pseudo-training/OOB arrangements from one small dataset.

It does not solve a nonrepresentative original sample. If important subpopulations are absent, bootstrap cannot invent them.
Variance

Averaging many repetitions reduces Monte Carlo variability of the bootstrap estimate, although resamples are dependent through the same dataset.

\(E_0\) bias

Fewer unique training cases can make bootstrap models worse than a full-data target model.

.632 correction

Balances pessimistic OOB behavior against optimistic re-substitution behavior.

A one-shot bootstrap repetition is not automatically better than hold-out. The bootstrap method is the repeated procedure plus its interpretation/correction.

Randomization and permutation

Randomization over labels

Shuffle labels to destroy the feature–label relationship. Ask whether the real score is much better than scores obtainable under chance assignments.

Randomization over samples/order

Reorder samples or alter split assignments to assess estimate stability. Pure row order should not affect deterministic batch learners, but can matter for online learning, SGD, order-sensitive preprocessing, or fold construction.

Permutation/randomization belongs partly to significance testing rather than plain point error estimation. Its role in the tutorial’s hierarchy is as a resampling-based evaluation tool.

Temporal, grouped and spatial validation

The correct sampling unit is determined by deployment—not by convenience.

Time split

Train on past, test on future. Estimates forecasting/generalization through time.

Group/person split

Keep all observations from one patient/user/subject together. Estimates unseen-group generalization.

Location split

Hold out whole regions/sites. Estimates generalization to unseen locations and reduces spatial leakage.

These are not automatically “OOD.” If future patients/regions are part of the intended target population, the split may simply be a more faithful in-distribution deployment simulation. It becomes distribution-shift/OOD evaluation when the held-out groups have mechanisms or characteristics not represented in training.

Leakage laboratory: overlapping time windows

Your example contains two related problems: future leakage and dependence from overlapping windows.

Future leakage: if a training example uses observations at or after the test target time, the model has information unavailable at the real prediction moment.
Overlap dependence: even without a future target crossing, neighboring rolling windows can share most timestamps. Randomly placing them into train and test makes the test artificially similar to training.
Safe rule: for test target \(t\), every feature used in training/evaluation must be information that would have been available before \(t\), and folds should respect temporal blocks/gaps when overlapping windows create dependence.

Interactive method selector

Choose the deployment/data constraints. The recommendation is a starting point, not an automatic law.

Criteria in detail

Domain / dependence

Choose splits that preserve the real independence structure. Random splitting is inappropriate when rows share patients, times, devices, or locations.

Dataset size

Large data can afford untouched hold-out. Small data pushes toward CV/bootstrap because a fixed test set consumes scarce training information.

Bias–variance

Small training folds may pessimistically evaluate the final model; small test folds increase variance of the estimate.

Expected true error

Very low error requires enough test cases to observe failures. Zero observed errors does not imply zero true risk.

Algorithm type

Duplicates act as weights in bootstrap. Order-sensitive algorithms can react to permutations. Expensive algorithms restrict repetition count.

Computational cost

Hold-out fits once, \(k\)-fold fits \(k\) times, 10×10 fits 100 times, LOO fits \(n\) times, bootstrap fits \(B\) times.

Exam insights and common traps

Trap 1: “sample”

Name whether you mean observed dataset, fold, bootstrap dataset, test sample, or one draw.

Trap 2: CV models

\(k\)-fold CV fits \(k\) models; it does not test one model \(k\) times.

Trap 3: estimator target

Call something optimistic/pessimistic only relative to a stated target.

Trap 4: variance

Small test sets increase variance of the error estimator, not necessarily model variance.

Trap 5: stratification

Preserving class proportions does not prevent patient/time/location leakage.

Trap 6: bootstrap

\(n\) bootstrap positions do not mean \(n\) unique observations.

Trap 7: confidence intervals

Hold-out has a clean fixed-model Bernoulli interpretation; CV fold dependence makes simple rigorous CIs harder.

Trap 8: OOD language

Unseen group is not automatically OOD; define the target population and deployment question.

Trap 9: temporal windows

Overlapping windows create dependence even when the exact window is absent from training.

Strong oral-exam answer structure: (1) state the target population and deployment scenario; (2) name the independent unit; (3) describe how train/test sets are formed; (4) state how many models are trained; (5) define the estimator and its target; (6) distinguish estimator bias, estimator variance, model bias, and model variance; (7) mention leakage/dependence and computation.
