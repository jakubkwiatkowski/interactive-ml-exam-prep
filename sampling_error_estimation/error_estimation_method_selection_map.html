How to interpret the axes

Dataset-size suitability

Hold-out becomes attractive when both train and test can be large. CV/bootstrap use small datasets more efficiently. LOO is usually practical only when n is small-to-moderate or training has a shortcut.

Dependence compatibility

Random IID splitting is inappropriate when rows share time, patients, groups, machines, or locations. Then use blocked/group-aware variants.

Low compute cost

Hold-out fits once; k-fold fits k times; 10×10CV fits 100; LOO fits n; bootstrap fits B.

Training-data efficiency

LOO and high-k CV train on almost all data. Hold-out sacrifices a fixed part. Bootstrap has n positions but only about 0.632n unique training cases.

Clean uncertainty

Independent hold-out has a clean fixed-model Bernoulli/binomial interpretation. CV fits different, dependent models, so simple confidence intervals are harder.

Partition stability

Repeated CV and repeated hold-out reduce dependence on one arbitrary partition, but they still reuse one dataset.
