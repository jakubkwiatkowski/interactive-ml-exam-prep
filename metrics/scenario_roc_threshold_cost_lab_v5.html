Ranking / top-k productivity and lift

Precision@k = how trustworthy the top-k alerts are. Gain@k = how many of all positives you captured. Lift@k = enrichment relative to random selection.

How to compute the optimal threshold efficiently

For each distinct score threshold \(t\), compute \[ EC(t)=P(+)C_{FN}FNR(t)+P(-)C_{FP}FPR(t) \] and choose \[ t^*=\arg\min_t EC(t). \] If class importance and error costs are equal, this reduces to maximizing Youden \(J=TPR-FPR\).
Practical implementation: sort all examples by score once; sweep through the distinct score values; update TP and FP counts cumulatively; evaluate only where the confusion matrix changes.

Cost and threshold — intuitive guide

ROC asks: what TPR/FPR trade-offs are available?
PR asks: how trustworthy are positive alerts as recall changes?
Cost curves ask: for which prevalence-and-cost conditions is each threshold cheapest?

A common horizontal coordinate is the probability-cost function: \[ PCF=\frac{P(+)C_{FN}}{P(+)C_{FN}+P(-)C_{FP}}. \] Near \(PCF=0\), false-positive burden dominates. Near \(PCF=1\), false-negative burden dominates. The vertical axis is normalized expected cost: \[ NEC=PCF\cdot FNR+(1-PCF)\cdot FPR. \] For one threshold, FNR and FPR are fixed, so \(NEC(PCF)\) is a straight line. Many thresholds give many lines; the lower envelope is the cheapest threshold for each operating condition.

ROC crossing only says that neither classifier dominates everywhere. Cost curves add the missing decision context and show where one threshold or classifier is preferable.
